# Règles projet — SaaS Bureau Parlementaire Lionel Tivoli

## Qualité et Méthode Antigravity

Pour chaque phase de ce projet, l'agent Antigravity doit rigoureusement appliquer cette méthode :
1. Lire le dépôt.
2. Identifier la stack.
3. Lire le README.
4. Lire les modèles, migrations, routes, contrôleurs et composants existants.
5. Produire un Artifact "Plan d'implémentation".
6. Produire un Artifact "Liste des fichiers à modifier".
7. Attendre validation si la tâche est sensible.
8. Implémenter par petites étapes.
9. Lancer les tests disponibles.
10. Lancer le lint si disponible.
11. Lancer le build si disponible.
12. Produire un Artifact final avec le résumé des changements, fichiers, commandes, résultats de test, limites et suite.

## Interdictions Strictes
- Ne jamais tout réécrire sans nécessité.
- Ne jamais supprimer définitivement des données métier par défaut (Archivage logique requis).
- Ne jamais mettre de vraies données personnelles dans les tests.
- Ne jamais stocker de secrets dans le dépôt.
- Ne jamais exposer de notes internes dans les logs.
- Ne jamais ajouter de dépendance lourde sans justification.
- Ne jamais lancer de commande destructive sans validation humaine.

## Sécurité / RGPD Obligatoires
Le logiciel traite des données personnelles sensibles dans un contexte politique.
- Mots de passe hashés et sessions sécurisées.
- HTTPS en production, protection CSRF si applicable.
- Validation serveur systématique de toutes les entrées.
- Contrôle des permissions rigoureux.
- Journalisation (AuditLog) des actions sensibles.
- Archivage logique au lieu de suppression définitive.
- Exports réservés aux utilisateurs autorisés.
- Logs exempts de données sensibles.
- Sauvegardes et politiques de conservation documentées.

## Qualité Attendue
- Code clair, architecture maintenable.
- Interface et messages d'erreur en français uniquement.
- Responsive desktop/tablette.
- Tests sur fonctions critiques obligatoires.
- Le README doit être tenu à jour systématiquement à chaque fin de phase.
